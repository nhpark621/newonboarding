# PRD – 체험형 경쟁사 분석 대시보드 온보딩 프로세스 기획

## 1. 개요

- **목적**
  - 사용자가 우리 회사의 주요 서비스를 간단히 체험할 수 있도록, Replit을 활용해 ‘회원가입 → 간단 온보딩 → 체험 페이지’까지 연결되는 프로토타입 웹페이지를 구현함
  - Lean MVP를 넘어 실제 고객 반응을 테스트하는 Superior MLP 개념으로 활용

- **타겟 사용자**
  - 회사 서비스에 관심은 있으나 아직 이용하지 않은 신규 잠재 고객

---

## 2. 문제 정의

- 기존 PDF 샘플 레포트 방식의 전환률이 낮고, 이후 유의미한 액션(문의, 미팅 등)으로 이어지지 않음
- 간편한 체험 경로가 없어, 제품의 실질적 효용을 전달하지 못하고 있음

---

## 3. 목표 및 성공 지표

- 5분 이내 온보딩 프로세스 체험 완료 가능하도록 구성
- 첫 방문자의 50% 이상이 온보딩 과정을 완료하는 것이 목표

---

## 4. 온보딩 구성 단계

### STEP 1. 경쟁사에 대해 궁금한 정보 입력

- **목적**: 사용자의 경쟁사에 관한 궁금한 점을 파악하여 이후 추천 서비스에 반영
- **화면 구성**
  - 헤더: "경쟁사와 관련해 가장 궁금한 점은 무엇인가요?"
  - 서브 문구: "입력하신 내용을 바탕으로 경쟁사 분석 결과를 더욱 정교하게 보여드립니다."
  - 입력창: 자유 입력 + 추천 태그 버튼 제공 (예: 신제품 출시 현황, 광고 발행, HR 인재채용, SNS 게시물 분석)
  - 자동 문장 생성: 태그 클릭 시 자연어 문장 자동 완성
  - 전송 버튼 클릭 시 Step 2로 이동

### STEP 2. 서비스 추천 및 선택

- **목적**: step1에서 사용자가 입력한 경쟁사 관련해서 궁금한 점으로 최적의 분석 서비스를 추천하고 선택 유도
- **구성**
  - 최대 4개 분석 서비스 카드 제공 (자동 추천 + 사용자 수정 가능)
  - 카드 구성: 서비스명, 설명, 컬러 인디케이터, Hover 시 미리보기 인터랙션
  - 예시 서비스명:
    - 뉴스·보도자료 분석
    - 신제품·서비스 출시
    - 인재 영입
    - 광고 분석
    - SNS 콘텐츠
    - 이벤트 모니터링
  - 서비스 상세 설명: 
    - 뉴스 보도자료 분석: 경쟁사의 최신 뉴스를 분석해 핵심 이슈와 전략 방향을 파악할 수 있어요. 주요 기사도 함께 추천드려요
    - 신제품 서비스 출시: 경쟁사가 출시한 신제품과 서비스, 기능 확장 현황을 한눈에 정리해드려요. 시장 대응 흐름을 빠르게 파악할 수 있어요
    - 인재 영입: 경쟁사의 채용 공고를 모니터링해 주요 영입 분야와 전략 변화 가능성을 알려드려요. 인재 움직임으로 준비 중인 전략을 유추할 수 있어요
    - 광고 분석: 경쟁사가 언제, 어디에, 어떤 광고를 집행했는지 확인하고, 신규 콘텐츠도 함께 분석해드려요. 광고 전략의 흐름을 쉽게 파악할 수 있어요
    - SNS 콘텐츠: 경쟁사의 SNS 채널에서 어떤 콘텐츠가 올라오고, 반응은 어떤지 실시간을 분석해드려요. 채널별 성과 비교도 가능해요
    - 이벤트 모니터링: 경쟁사가 진행한 프로모션, 할인, 사은품 증정 등 이벤트 현황을 정리해드려요. 혜택 구성과 타이밍까지 비교할 수 있어요
  - "분석하기" 클릭 시 Step 3로 이동

### STEP 3. 회원가입 및 추가 정보 입력

- **목적**: 서비스 체험 전 사용자 정보 수집을 통해 개인화된 분석 결과 제공
- **화면 구성**
  - 필수 입력: 이메일, 비밀번호, 회사명, 소속 팀, 담당 제품/서비스
  - UI 요소: 유효성 검사, 조건 충족 시 체크 아이콘 표시
  - 가입 완료 후 "경쟁사 분석 시작하기" 버튼 → 체험형 분석 대시보드 이동
  → 현재는 대시보드 구현 전이므로, 임시 페이지(`dashboard.html`)로 리디렉션 처리
---

## 5. 기타 참고사항

- **기술 스택**
  - 프론트엔드: React or HTML 기반 Replit 프로젝트
  - 사용자 입력 기반 추천 로직에 OpenAI API 연동
  - 태그 기반 자동 문장 입력 및 추천 카드 구성은 JavaScript 동적 처리

- **UX 고려사항**
  - 각 단계는 직관적이고 시각적으로 매끄럽게 연결되도록 구성
  - CTA 버튼(예: 분석하기)은 화면 하단 고정, 클릭 유도 강함

---

## 6. 요약 동작 플로우

1. Step 1: 고민 입력 → 자동 문장 생성 or 직접 입력 → Step 2 이동
2. Step 2: 추천 서비스 확인 및 수정 → 분석하기 클릭 → Step 3 이동
3. Step 3: 회원가입 및 정보 입력 → 대시보드 이동 후 서비스 체험 시작

---

## 7. 예상 기대 효과

- 체험 기반의 인터랙션을 통해 사용자의 '아하 모먼트'를 빠르게 유도
- 제품 가치에 대한 직접 체감을 통해 전환률 상승 기대
- 고객 반응 데이터 확보 및 실사용 경험 기반 피드백 수집 가능



## 8. 이벤트 트래킹 설계 (퍼널 분석 목적)
- 목적: 온보딩 흐름에서 사용자 이탈 지점을 파악하고 전환 개선을 위한 퍼널 분석 기반 확보
- 활용 도구: GA4 또는 자체 트래킹 함수(trackEvent) 활용

[이벤트 항목 목록]
- input_started
: Step 1에서 사용자가 텍스트 입력을 시작할 때
: 파라미터 – method: 'manual' 또는 'tag'

- tag_selected
: 태그를 클릭했을 때
: 파라미터 – tag_name: '신제품 출시'

- auto_text_generated
: 태그 클릭으로 자동 문장이 생성되었을 때
: 파라미터 – generated_text: '경쟁사에서 신제품을 출시한 것 같아요'

- step1_completed
: 고민 입력을 완료하고 Step 2로 이동했을 때
: 파라미터 없음

- card_shown
: 추천 카드가 화면에 나타났을 때
: 파라미터 – count: 4

- card_clicked
: 사용자가 추천 카드 중 하나를 선택했을 때
: 파라미터 – selected_card: '광고 분석'

- signup_started
: Step 3의 회원가입 화면에 진입했을 때
: 파라미터 없음

- signup_completed
: 회원가입을 완료했을 때
: 파라미터 없음

- onboarding_completed
: Step 3까지 온보딩 전 과정을 완료했을 때
: 파라미터 없음
