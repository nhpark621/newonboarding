# Events Monitoring — MVP Spec

## Context
Dashboard URL: https://onboarding-flow-sparktest.replit.app/dashboard

We will implement the six competitor‑analysis items on the dashboard one by one. Start with **Events Monitoring**.

## What to Build (MVP)
- When the user selects **“Events Monitoring”** on the **[Recommend a tailored analysis service]** page, render the dedicated module.
- Read `company`, `team`, `productOrService`, and `competitors[]` (use a skip flag if empty) from the **[Enter competitor analysis info]** page.
- If competitors were skipped, populate defaults with `mockPickCompetitors(productOrService)`.

### Channel → Event Pages → Event Items (last 30 days)
- Recommend channels with industry rules and probe common paths: `/event | /promotion | /benefit | /campaign | /sale | /notice | /news`.
- From listing/detail pages, extract event links and **store only items posted/active within the last 30 days**.

## Storage & APIs
- **DB:** SQLite (Prisma) or JSON fallback

**Endpoints**
```
POST /api/channels/suggest
POST /api/events/discover
POST /api/events/crawl   (default: limitDays=30)
GET  /api/events/summary?companyId=...
```

## UI (Minimal)
- Header: **“Events (last 30 days)”**
- Badges: number of channels, discovered links, new events, ending soon (≤ 7 days)
- Two tables:
  - **“New in 30d”**
  - **“Ending soon”** (title / channel / period / source link)

## Files to Add
- `services/industryMap.(ts|js)`
- `services/channelSuggest.(ts|js)`
- `services/linkDiscover.(ts|js)`
- `services/eventExtract.(ts|js)`
- `services/dedupe.(ts|js)`

## Guardrails
- Respect robots.txt and Terms; if behind login, **store links only**; limit infinite scroll to the first 1–2 pages.
- Keep other items (News/SNS, etc.) as **stubs only**—do not implement them now.

## Acceptance
- Selecting **“Events”** in **[Recommend a tailored analysis service]** renders real data in **[Competitor Analysis Dashboard]**.
- On first run, channels/links/events are saved with the **last‑30‑days filter** applied.
- After refresh, both the summary and lists reload from storage correctly.
