# API Spec – 경쟁사 고민 기반 추천 서비스 매핑

본 문서는 체험형 온보딩 웹페이지에서 사용자의 입력을 기반으로 추천 분석 서비스를 제공하기 위한 OpenAI API 연동 사양을 정의합니다.

---

## 🔗 목적

사용자가 입력한 경쟁사 관련 고민 문장을 바탕으로, 적절한 분석 서비스 항목을 자동으로 추천합니다.

---

## 🔧 API 호출 사양

- **API 이름**: `POST /api/recommend-services`
- **호출 방식**: OpenAI GPT API 사용 (gpt-4o 또는 gpt-4-turbo)
- **입력 파라미터**
```json
{
  "user_input": "경쟁사의 신제품 출시 전략이 궁금해요"
}
```
- **응답 예시**
```json
{
  "recommended_services": [
    "신제품·서비스 출시",
    "광고 분석",
    "SNS 콘텐츠"
  ]
}
```

---

## 📌 처리 흐름

1. 사용자가 Step 1 화면에서 고민을 입력하거나 태그를 선택함  
2. 해당 텍스트를 OpenAI API로 전송  
3. 추천 서비스 리스트가 JSON 형태로 응답됨  
4. 응답 결과를 바탕으로 Step 2에서 분석 카드 자동 구성

---

## 🧠 프롬프트 기본 구조 (예시)

```
사용자가 다음과 같은 고민을 입력했습니다:
"경쟁사의 SNS 전략이 요즘 눈에 띕니다."

이 고민과 관련해 추천할 만한 경쟁사 분석 서비스 항목 3~4가지를 제안해 주세요.  
각 항목은 10자 이내의 키워드 형태로 출력해 주세요.
```

---

## 🛑 주의 사항

- 응답 항목은 반드시 내부에서 정의된 6가지 분석 항목 중에서 선별  
  - 뉴스·보도자료 분석  
  - 신제품·서비스 출시  
  - 인재 영입  
  - 광고 분석  
  - SNS 콘텐츠  
  - 이벤트 모니터링
- 잘못된 입력 또는 공백 입력 시 기본 추천 3개 반환
